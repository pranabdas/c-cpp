Graphs

Graphs are everywhere around us: computer networks, road networks, social networks, electronic circuits, molecular structure etc.

Terminology:

G=(V,E)G = (V, E)G=(V,E)

V⇒VerticesV \Rightarrow \text{Vertices}V⇒Vertices

E⇒Edges⊆V×VE \Rightarrow \text{Edges} \subseteq V \times VE⇒Edges⊆V×V

Edges can be directed or undirected. Say we have two vertices vvv and www. Possible Edges could be e=(w,v)e = (w, v)e=(w,v) or e′=(v,w)e' = (v, w)e′=(v,w). In case of undirected edges, e={v,w}e = \{v, w\}e={v,w}. We will use the set notation where order does not matter.

In case of undirected graph, number of edges:

∣E∣≤(∣V∣/22)=O(∣V∣2)|E| \le \binom{|V|/2}{2} = \mathcal{O}(|V|^2)∣E∣≤(2∣V∣/2​)=O(∣V∣2)

If we have directed graph, there will be factor of 2, still O(∣V∣2)\mathcal{O}(|V|^2)O(∣V∣2).

Graph representation​

How can we store a graph in the computer? Few of the options are:

  • Edge list
  • Adjacency list
  • Adjacency matrix

Adjacency matrix: say we have a graph G=(V,E)G = (V, E)G=(V,E) with nnn vertices and mmm edges. We can represent GGG using and n×nn \times nn×n matrix MMM, where element M[i,j]=1M[i, j] = 1M[i,j]=1 if (i,j)(i, j)(i,j) is an edge of GGG, and 000 if it is not an edge. This representation allows faster look up, and updates for edge insertion and deletion. However, it may use excessive space for sparse graphs.

Adjacency list: We can more efficiently store sparse graphs by using linked lists to store neighbors of each vertex.

Sparse vs. dense: usually sparse graphs number of edges ∣E∣≈O(∣V∣)|E| \approx \mathcal{O}(|V|)∣E∣≈O(∣V∣), while for dense graphs ∣E∣≈O(∣V∣2)|E| \approx \mathcal{O}(|V|^2)∣E∣≈O(∣V∣2).
